**Integration checker concerns addressed:**

1. **INST-02 depmod (raised as critical):** False positive. In debhelper compat 13, `dh_installmodules` runs as part of the `dh $@` sequence and auto-detects `.ko` files installed in the package staging directory. No `debian/igh-seeedstudio.modules` file is needed for depmod generation — that file is for listing modules to load at boot, not for triggering depmod.

2. **ethercat.service unit (raised as moderate):** False positive. IgH's `make install DESTDIR=$(PKGDIR)` installs the `ethercat.service` unit into the staging directory. `dh_installsystemd` picks it up from there. No separate `debian/igh-seeedstudio.service` file is needed.

3. **Dockerfile --platform (raised as low):** Valid observation, classified as tech debt. On `ubuntu-22.04-arm` CI runner this is correct. Adding `--platform linux/arm64` would prevent silent amd64 builds on developer machines.
