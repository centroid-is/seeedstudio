**debian/control** — Two stanzas separated by exactly one blank line:

```
Source: igh-seeedstudio
Section: kernel
Priority: optional
Maintainer: SeeedStudio <support@seeedstudio.com>
Build-Depends: debhelper-compat (= 13),
               build-essential,
               autoconf,
               automake,
               libtool,
               nvidia-l4t-kernel-headers
Standards-Version: 4.6.1

Package: igh-seeedstudio
Architecture: arm64
Depends: ${misc:Depends}, kmod
Description: IgH EtherCAT Master for SeeedStudio Jetson
 Pre-compiled IgH EtherCAT Master 1.6 with the r8169 native driver
 for NVIDIA Jetson platforms running Tegra L4T kernel 5.15.148.
 Includes ec_master.ko and ec_r8169.ko kernel modules, the ethercat
 CLI tool, and systemd service integration.
```

Key details:
- `nvidia-l4t-kernel-headers` without version pin (exact apt version string will be refined in Phase 2 when build environment is set up — per research recommendation)
- Build-Depends continuation lines indented with spaces to align
- Long description lines start with single space; no trailing whitespace on blank separator line between stanzas
- `${misc:Depends}` is a debhelper substitution variable resolved at build time
- `kmod` provides modprobe/depmod needed by postinst in Phase 3

**debian/rules** — Minimal debhelper Makefile:

```makefile
#!/usr/bin/make -f
export DH_VERBOSE = 1

%:
	dh $@
```

CRITICAL: The line `	dh $@` MUST use a hard tab character (ASCII 0x09), NOT spaces. After writing the file, run `chmod +x debian/rules` to make it executable. Verify with `test -x debian/rules`.

**debian/changelog** — Strict dpkg-parsechangelog format:

```
igh-seeedstudio (1.6.0) unstable; urgency=medium

  * Initial release.
  * IgH EtherCAT Master 1.6 for Jetson aarch64 with r8169 native driver.

 -- SeeedStudio <support@seeedstudio.com>  Mon, 17 Mar 2026 12:00:00 +0000
```

Critical formatting rules:
- Package name `igh-seeedstudio` must exactly match `Source:` in debian/control
- Two spaces before each `*` bullet
- One space before `--` on the maintainer line
- Exactly two spaces between closing `>` and the date
- Date is RFC 5322 format
- Version `1.6.0` without Debian revision suffix (native package)
