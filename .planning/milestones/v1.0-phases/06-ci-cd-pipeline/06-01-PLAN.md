  <action>
Create directory .github/workflows/ and create .github/workflows/build.yml with:

WORKFLOW STRUCTURE:
- name: "Build .deb"
- on:
  - push to branches: [main]
  - push tags: ["v*"]

JOB 1 — "build":
- runs-on: ubuntu-22.04-arm (native arm64, NOT QEMU)
- steps:
  1. actions/checkout@v4
  2. Run "docker build -t igh-seeedstudio-build ." to build the .deb inside Docker (leverages existing Dockerfile which already does the full build pipeline including L4T repo setup, kernel headers extraction, dpkg-buildpackage, dpkg -i verification, and ec_r8169.ko assertions)
  3. Extract the .deb from the Docker image:
     docker create --name extract igh-seeedstudio-build
     docker cp extract:/build/igh-seeedstudio_1.6.0_arm64.deb .
     docker rm extract
  4. Upload .deb as a workflow artifact using actions/upload-artifact@v4 with name "igh-seeedstudio-deb" and path "igh-seeedstudio_1.6.0_arm64.deb"

JOB 2 — "release":
- runs-on: ubuntu-22.04-arm
- needs: [build]
- if: startsWith(github.ref, 'refs/tags/v')
- permissions:
    contents: write
- steps:
  1. actions/checkout@v4
  2. Download the .deb artifact using actions/download-artifact@v4 with name "igh-seeedstudio-deb"
  3. Create GitHub Release and upload .deb:
     gh release create "$GITHUB_REF_NAME" igh-seeedstudio_1.6.0_arm64.deb --title "$GITHUB_REF_NAME" --generate-notes
     env:
       GH_TOKEN: ${{ github.token }}

IMPORTANT DETAILS:
- Do NOT use QEMU or cross-compilation. The runner "ubuntu-22.04-arm" is native arm64.
- The Dockerfile already handles the ENTIRE build pipeline. The workflow just runs "docker build" and extracts the artifact.
- The .deb path inside the container is /build/igh-seeedstudio_1.6.0_arm64.deb (dpkg-buildpackage outputs to parent of WORKDIR).
- Use ${{ github.token }} for the GH_TOKEN (automatic GITHUB_TOKEN, no PAT needed).
- gh release create uses --generate-notes to auto-generate release notes from commits.
  </action>
